# Security Policy

## Supported Versions

SROS v1 uses a single hardened line. Security fixes are applied to the latest `main` release line only.

## Reporting a Vulnerability

Report vulnerabilities privately to: `security@platxp.example`.

Please include:
- Affected version and commit hash
- Reproduction steps
- Expected vs actual behavior
- Impact assessment

Do not open public issues for suspected vulnerabilities until triage is complete.

## Response Targets

- Initial acknowledgement: within 3 business days
- Triage decision: within 7 business days
- Remediation ETA: shared after triage
